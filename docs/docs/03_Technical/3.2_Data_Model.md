# 3.2 Data Model and Persistence

The system of record is **PostgreSQL** via **Prisma**; **Redis** is a best-effort read cache and rate-limit/session store. This doc describes the schema, the relationships, the caching strategy, and the currency units — the things you must get right when touching persistence.

## Entity Map

```
User ──1:N── Save ──1:N── ClubStint ──1:N── Player        (Player.activeClubStint, nullable)
 │            │              │          └─1:N─ PlayerSeasonStats ─N:1─ Player
 │            │              ├─1:N── TeamSeasonStats ─N:1─ Competition
 │            │              ├─1:N── Trophy            ─N:1─ Competition
 │            │              └─1:N── Transfer  (also ─N:1─ Player, nullable)
 │            ├─1:N── ScoutPlaybook
 │            ├─1:N── ShortlistItem ─N:1─ Fc26Player
 │            └─1:N── SavedSearch
 ├──1:N── Session
 └──1:N── Account

Player ──1:N── PlayerOvrHistory
Fc26Player  (standalone read-only dataset; only ShortlistItem references it)
Competition (standalone reference data)
```

Better Auth owns `User`, `Session`, `Account`, `Verification`.

## Core Tables

### `User`
Better Auth user. `email` unique; `role` (`user`/`admin`); `plan` (`UserPlan`, default `FREE`); ban fields (`banned`, `banReason`, `banExpires`). Deleting a user cascades to `Save`, `Session`, `Account`.

### `Save`
One Career Mode career. `currentYear` (Int, col default 2026), `currentSeason` (String `"YYYY/YY"`, col default `"2026/27"` — **but `createSave` overrides to 2025 / `"2025/26"`**), `budget` and `balance` (Float, millions of €). Owns stints, players, transfers, playbooks, shortlist, saved searches — all `onDelete: Cascade`.

### `ClubStint`
A spell at one club within a save. `club` (string, validated against the in-memory club list), `startYear`/`endYear` (strings), `isCurrent` (exactly one `true` per save). Indexed `@@index([saveId])`, `@@index([saveId, isCurrent])`. Owns the stint's players (via `activeClubStintId`), team stats, trophies, transfers, player season stats.

### `Player`
A squad member within a save. `position` (`Position`), `status` (`PlayerStatus`), `age`, `ovr`, optional `potential`, `shirtNumber`, `nation`, `salary` (thousands), `marketValue` (millions). `activeClubStintId` nullable — null means released/loaned-out. `alternativePosition` is a JSON `{ positions: Position[] }`. Indexed `@@index([saveId])`, `@@index([saveId, activeClubStintId])`.

### `PlayerSeasonStats`
Per (player, stint, season) — `@@unique([playerId, clubStintId, season])`. `goals, assists, matches, yellowCards, redCards, cleanSheets`. `goalContributions` is **not** a column — it is derived (`goals + assists`) in every projection.

### `PlayerOvrHistory`
Per (player, season) snapshot — `@@unique([playerId, season])`. `ovr`, `marketValue`. Written on season advance; read to compute `ovrDelta`/`marketValueDelta` on active-player listings.

### `TeamSeasonStats`
Per (stint, competition, season) — `@@unique([clubStintId, competitionId, season])`. `goalsPro, goalsAgainst, wins, draws, losses`, `leaguePosition` (leagues), `cupResult` (cups/European). `competitionId` is nullable for legacy rows; current code always sets it.

### `Trophy`
Per (stint, competition, year) — `@@unique([clubStintId, competitionId, year])`. Created on season advance (auto, for wins) or manually.

### `Transfer`
`type` (`TransferType`), `playerName`, `from`, `to`, `season`, optional `fee` (millions), nullable `playerId` and `clubStintId`. Indexed `@@index([saveId])`, `@@index([saveId, season])`, `@@index([clubStintId, season])`.

### `Competition`
Reference data. `name` unique, `type` (`CompetitionType`), optional `country`. Seeded; cached 24h.

### `Fc26Player`
The read-only scouting dataset (~one row per real player). `sofifaId` unique, `positions` (String[]), `ovr`, `potential`, `marketValue` (millions), `wage` (thousands), `nation`, `club`, `league`, and ~50 detailed attributes (pace/shooting/…, plus per-skill columns). Heavily indexed for filtering:
- B-tree: `ovr`, `potential`, `age`, `nation`, `league`, `pace`, `preferredFoot`.
- **GIN**: `positions`, `playerTags`, `playerTraits` (array containment for `hasSome` filters).

### `ScoutPlaybook` / `ShortlistItem` / `SavedSearch`
Per-save scout state. `ScoutPlaybook` (`weights` JSON, `preferences` JSON, `isDefault`, `@@unique([saveId, name])`). `ShortlistItem` (`@@unique([saveId, fc26PlayerId])`, `priority` `ShortlistPriority?`, `notes`). `SavedSearch` (`filters` JSON, `@@unique([saveId, name])`).

## Currency Units

| Field(s) | Unit | Example |
|---|---|---|
| `Player.salary`, `Fc26Player.wage` | thousands of € | `75` = €75K |
| `Save.budget`, `Save.balance`, `Player.marketValue`, `Transfer.fee` | millions of € | `100` = €100M, `0.9` = €900K |

Services return both the raw number and a `*Formatted` string (`balanceFormatted`, `marketValueFormatted`, `salaryFormatted`, `feeFormatted`) via `formatMarketValue` / `formatBalance` / `formatSalary` in `shared/utils/currency.ts`.

## Prisma Client

`shared/lib/prisma.ts` exports a singleton. `DATABASE_URL` is rewritten at startup to append `connection_limit=10&pool_timeout=20`. `DIRECT_URL` (Prisma `directUrl`) is the un-pooled Neon connection used only by migrations (bypasses PgBouncer). Query logging is on in dev only. The instance is memoized on `globalThis` only in non-production (avoids exhausting connections during `tsx watch` reloads).

> Drift note: `CLAUDE.md` says `connection_limit=20` and names the env var `directUrl`; the code uses `10` and `DIRECT_URL`. Trust the code.

## Redis Caching Strategy

`shared/utils/cache.ts` provides `cacheGet` / `cacheSet(key, value, ttl)` / `cacheInvalidate(...keys)` / `cacheInvalidatePattern(glob)` (SCAN-based). **All swallow errors** — a Redis outage degrades to direct-Postgres reads, never a failure.

### Key scheme & TTLs

| Key | Holds | TTL |
|---|---|---|
| `session:<token>` | resolved Better Auth session | 5 min |
| `mcp:session:<token>` | `{ userId }` for MCP | 5 min |
| `user:<userId>:saves` | save list | 15 min |
| `save:<saveId>` | save detail | 30 min |
| `save:<saveId>:players[:active[:<season>]]`, `:players:loaned` | squad listings | 30–60 min |
| `save:<saveId>:player:<playerId>` | player detail | 1 h |
| `save:<saveId>:transfers[:current]` | transfers | 30 min |
| `save:<saveId>:team-stats[:<season>]` | team stats | 1 h |
| `save:<saveId>:trophies` | trophies | 1 h |
| `save:<saveId>:club-stints` | stints | 1 h |
| `fc26:list:<json>`, `fc26:detail:<id>`, `fc26:filters` | dataset | 24 h |
| `fit-score:<club>:<posGroup>:<objective>:<sofifaId>` | fit score | 6 h |
| `competitions:all`, `competitions:european`, `competitions:country:<c>:ids` | reference | 24 h |
| `chat:ratelimit:<userId>`, MCP rate-limit keys | counters | window-scoped |

### Invalidation patterns
- **Point**: writes call `cacheInvalidate` with the specific keys (e.g., transfer create clears `transfers`, `transfers:current`, `players:loaned`).
- **Pattern**: cascades (season advance, save delete) call `cacheInvalidatePattern('save:<id>:*')`.
- **Scope**: each write invalidates the keys it affects; season-scoped and list keys are invalidated alongside the base resource.

## Migrations & Seeds

- `prisma/migrations/` — versioned SQL; applied in prod by Railway's `preDeployCommand` (`prisma migrate deploy`).
- `prisma/seed.ts` (clubs/base), `prisma/seed-competitions.ts` (competitions), `scripts/seed-fc26.ts` (dataset), `prisma/migrate-data.ts` (one-off data migration).
- `scripts/*fit-score-club-alias*` — generate/sync the club-name aliases used to map internal club names to the fit-score service's expectations.
