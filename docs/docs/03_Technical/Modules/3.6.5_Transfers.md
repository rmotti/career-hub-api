# 3.6.5 Transfers

## Responsibility

Registers incoming/outgoing transfers and loans. Each write is transactional: it links or creates a `Player`, updates the player's squad membership, and adjusts the save's `balance`. Transfers are scoped to the club stint that recorded them.

## Files

```
features/transfers/
├── transfers.routes.ts
├── transfers.controller.ts
└── transfers.service.ts   listTransfers, createTransfer, updateTransfer, deleteTransfer
```

## Endpoints

| Method | Path | Notes |
|---|---|---|
| GET | `/api/saves/:saveId/transfers` | `?season=current` → current stint + current season only. |
| POST | `/api/saves/:saveId/transfers` | Returns `{ transfer, playerId, save }`. |
| PUT | `/api/saves/:saveId/transfers/:tid` | Re-applies the balance delta on fee/type change. |
| DELETE | `/api/saves/:saveId/transfers/:tid` | 204. |

## Data Flow — Create (`createTransfer`)

All in one transaction:

1. Validate `from`/`to` present and `season` matches `^\d{4}\/\d{2}$`.
2. For **outgoing** (`venda`/`emprestimo_saida`) with a `playerId`: assert the player exists and is in the active squad.
3. For **incoming** (`compra`/`emprestimo_entrada`): resolve the player —
   - matching `playerId` → re-attach; else
   - inactive same-name player → reactivate; else
   - **create** a new `Player` with defaults (`position MEI`, `age 25`, `ovr 70`).
   New status: `Role` (purchase) or `Loan` (loan-in). Create the current-season stats row.
4. Create the `Transfer` (stamped with the current `clubStintId`).
5. For **outgoing**: detach the player (`activeClubStintId = null`; loan-out also sets `status Loan`).
6. **Balance**: only `compra` (subtract `fee`) and `venda` (add `fee`) move `balance`; loans don't. Returns the updated `save`.

Then invalidate `transfers`, `transfers:current`, `players:loaned`.

### Update
`updateTransfer` reverses the old fee's balance effect and applies the new one when `fee` or `type` changes (purchase/sale only), then updates the row.

## Local Domain Rules

1. Types: `compra, venda, emprestimo_entrada, emprestimo_saida`.
2. `fee` is millions of €; loans typically `0`/null and never touch the balance.
3. `playerId` is optional on create — incoming may auto-create; outgoing expects a match.
4. Transfers are stint-scoped: `?season=current` excludes transfers from a previous club in the same season (see `docs/changelog-transfers-club-stint-scope.md`).
5. ⚠️ **Ownership not enforced** — save looked up by id only. See [3.7 Security #1](../3.7_Security.md#1-broken-object-level-authorization-idor--high). Because this module mutates `balance` and squad membership, the IDOR here is write-impactful.

## Extension Points

- Transfer windows (summer/winter) grouping.
- Cumulative spend/income per season/stint.
- CSV import of a transfer batch.
- Surface `clubStintId` in history views (already on the response).
