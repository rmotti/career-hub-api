# 3.6.8 FC26 Dataset

## Responsibility

Serves the **read-only** FC26 player database used for scouting: rich attribute filtering, filter metadata for UIs, single-player lookup, and optional **fit-score** enrichment against a save's current club. This module is the data backbone of the entire Scout surface.

## Endpoints

| Method | Path | Notes |
|---|---|---|
| GET | `/api/fc26-players/filters` | `{ positions, nations, leagues, clubsByLeague }`. Cached 24h. |
| GET | `/api/fc26-players` | Filterable list → `{ players, total, limit, offset }`. |
| GET | `/api/fc26-players/:sofifaId` | Full player profile. |

## Filtering (`listFc26Players`)

- **Same-field = OR, cross-field = AND.** E.g. `positions=MC,ATA` → MC *or* ATA; `&leagues=Premier League` adds an AND.
- Direct filters: `positions` (array `hasSome`), `nations`, `clubs`, `leagues`, `preferredFoot`, `traits` (array `hasSome`), and min/max ranges on `ovr`, `age`, `potential`, `marketValue`, `pace`, `height`.
- `primaryPositions` / `secondaryPositions` are applied in memory after a `hasSome` pre-filter (primary = `positions[0]`, secondary = the rest).
- Pagination: `limit` (default 20, capped 100), `offset`. `total` reflects the filters.
- Ordered by `ovr desc`. The base (unenriched) result is cached 24h under `fc26:list:<json-of-filters>`.

## Fit-score enrichment

When a `saveId` is supplied and the save has a current club, the service enriches each player with `fitScore`, `fitConfidence`, `fitProfileSize`:

1. Group candidates by **position group** (`POSITION_GROUP` maps internal codes → `GK/CB/RB/.../CF`).
2. For each group, check per-player cache (`fit-score:<club>:<group>:<objective>:<sofifaId>`, 6h); for the rest, call `fetchFitScoreBatch` against `${FIT_SCORE_SERVICE_URL}/score/batch` with the club name, position group, objective, and candidate descriptors (age, nationality, origin league, market value in €, `fee_type: 'paid'`).
3. **3s timeout, fail-open**: any error/timeout/unset URL → no scores (the listing still returns). Successful scores are cached.
4. If any scores exist, re-sort by `fitScore desc` (nulls last).

Club/league/nationality names are translated to the fit-score service's vocabulary via `fit-score-maps.ts` + the generated alias table.

## Local Domain Rules

1. The dataset is **read-only** — never mutated outside the seed script (`scripts/seed-fc26.ts`).
2. `sofifaId` is the external stable id; `id` is the internal autoincrement. Lookups by `sofifaId`.
3. `marketValue` is millions of €, `wage` is thousands of € (matching the rest of the domain).
4. GIN indexes back the array `hasSome` filters (`positions`, `playerTags`, `playerTraits`); B-tree indexes back the scalar range filters.
5. Fit-score is best-effort: its absence degrades quality, never availability.
6. These endpoints are part of the paid (PRO) Scout surface.
