# 3.6.7 Competitions and Clubs

## Responsibility

Reference data used to seed and validate the rest of the domain: the list of **clubs** (by league and country) and the fixed set of **competitions** (leagues, cups, European). Clubs are in-memory in code; competitions live in Postgres and are cached.

## Clubs (in-memory)

`CLUBS_BY_LEAGUE` is a hardcoded map covering ~40 leagues across many countries (England, Spain, Germany, France, Italy, Portugal, Netherlands, Brazil, Argentina, MLS, Saudi, etc., plus a "No League" bucket for unlicensed/international sides). Derived structures:
- `CLUBS` ‚Äî flat list (`Object.values(...).flat()`).
- `LEAGUE_TO_COUNTRY` ‚Äî maps each league to a country (used to pick the country's competitions when creating a save/stint).
- Helpers: `clubExists(club)`, `findLeagueByClub(club)`, `getClubsByLeague(league)`.

This is the source of truth for club validation in `saves` and `club-stints`. Updating for a new season is a code edit + deploy.

## Competitions (DB + cache)

- `listCompetitions` ‚Üí cache `competitions:all` (24h), ordered by country/type/name.
- `listEuropeanCompetitions` ‚Üí cache `competitions:european` (24h) ‚Äî drives the season-rollover "which European competition?" dropdown.
- `getCompetitionIdsByCountry(country)` ‚Üí cache `competitions:country:<c>:ids` (24h) ‚Äî used by `saves`/`club-stints` to create per-competition `TeamSeasonStats`.

## Local Domain Rules

1. A club must exist in `CLUBS` to start a save or stint (else 400).
2. `Competition.name` is unique; `type ‚àà {League, NationalCup, SuperCup, EuropeanCup}`; `country` optional.
3. Competition data is static reference data ‚Äî read-only via the API, seeded via `prisma/seed-competitions.ts`.
4. These endpoints are save-independent and require only authentication (no ownership concern).

## Extension Points

- Move clubs to a seeded table (or JSON asset) so roster updates don't require a code change.
- Expose competitions filtered by country for richer client UIs.
- Add club crests / metadata.
