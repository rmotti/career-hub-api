# 3.6.7 Competitions and Clubs

## Responsibility

Reference data used to seed and validate the rest of the domain: the list of **clubs** (by league and country) and the fixed set of **competitions** (leagues, cups, European). Clubs are in-memory in code; competitions live in Postgres and are cached.

## Clubs (in-memory)

`CLUBS_BY_LEAGUE` is a hardcoded map covering ~40 leagues across many countries (England, Spain, Germany, France, Italy, Portugal, Netherlands, Brazil, Argentina, MLS, Saudi, etc., plus a "No League" bucket for unlicensed/international sides). Derived structures:
- `CLUBS` — flat list (`Object.values(...).flat()`).
- `LEAGUE_TO_COUNTRY` — maps each league to a country (used to pick the country's competitions when creating a save/stint).
- Helpers: `clubExists(club)`, `findLeagueByClub(club)`, `getClubsByLeague(league)`.

This is the source of truth for club validation in `saves` and `club-stints`. Updating for a new season is a code edit + deploy.

## Competitions (DB + cache)

- `listCompetitions` → cache `competitions:all` (24h), ordered by country/type/name.
- `listEuropeanCompetitions` → cache `competitions:european` (24h) — drives the season-rollover "which European competition?" dropdown.
- `getCompetitionIdsByCountry(country)` → cache `competitions:country:<c>:ids` (24h) — used by `saves`/`club-stints` to create per-competition `TeamSeasonStats`. The wiring is by **country** (`LEAGUE_TO_COUNTRY[league]`), not by league name.

### Seed coverage (F-005)

`prisma/seed-competitions.ts` derives its country list from `LEAGUE_TO_COUNTRY` and covers **every** country that has selectable clubs (24 countries: domestic league + national cup + super cup where one exists), plus the 4 UEFA cups (`country: null`). A coverage guard throws at seed time if any `LEAGUE_TO_COUNTRY` country lacks competitions, so a roster update can't silently re-open the gap; the same invariant is asserted in `features/competitions/__tests__/seed-competitions.test.ts`. The seed is idempotent (upsert on `Competition.name @unique`) and is **data-only** — safe to run against prod with no migration. Continental cups are **Europe-only** by decision; non-European continental competitions (Libertadores, AFC CL, …) are out of scope for F-005.

## Local Domain Rules

1. A club must exist in `CLUBS` to start a save or stint (else 400).
2. `Competition.name` is unique; `type ∈ {League, NationalCup, SuperCup, EuropeanCup}`; `country` optional.
3. Competition data is static reference data — read-only via the API, seeded via `prisma/seed-competitions.ts`.
4. These endpoints are save-independent and require only authentication (no ownership concern).

## Extension Points

- Move clubs to a seeded table (or JSON asset) so roster updates don't require a code change.
- Expose competitions filtered by country for richer client UIs.
- Add club crests / metadata.
