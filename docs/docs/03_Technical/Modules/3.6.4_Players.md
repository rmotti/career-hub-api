# 3.6.4 Players (Squad)

## Responsibility

Owns the squad: creating, editing, releasing players; per-season stats; OVR/market-value history; loaned-out players; and the bulk import of a club's real FC26 roster. This is the busiest CRUD module.

| Method | Path | Notes |
|---|---|---|
| GET | `/api/saves/:saveId/players` | Modes via query (see below). |

### List modes
- **none** ‚Üí all players with summed `totalStats`.
- **`?active=true`** ‚Üí current squad, sorted by position, with `currentSeasonStats`, `ovrDelta`, `marketValueDelta` (vs. last `PlayerOvrHistory`).
- **`?active=true&season=YYYY/YY`** ‚Üí historical squad: players who had stats that season in the current stint.
- **`?loaned=true`** ‚Üí players loaned out from the current stint, with `loanedTo` and `loanSeason`.

## Data Flow Highlights

- **Create**: validates `potential` (40‚Äì99), `shirtNumber` (1‚Äì99, uniqueness among active squad ‚Üí 409 `SHIRT_NUMBER_CONFLICT`), and `alternativePosition` (valid, non-duplicate, excludes primary). Links to the current stint and creates a `PlayerSeasonStats` row (with optional starting `matches`).
- **Update stats** (`PATCH .../stats`): only allowed for a player in the **active** squad of the **current** stint and season; rejects `goalContributions` as input (it's derived). `P2025` ‚Üí friendly 404.
- **Import** (`importFc26Squad`): pulls every `Fc26Player` whose `club` matches the current stint; skips names already in the save; default status `Important`; skips invalid dataset positions; returns `{ imported, skipped, total }`.
- **Release**: detaches the player (stays in the save for history) and invalidates the squad caches.

## Local Domain Rules

1. "Active" = `activeClubStintId === currentStint.id`. Released/loaned-out players are detached.
2. Shirt numbers unique among **active** players only.
3. `goalContributions = goals + assists`, always derived, never stored or accepted.
4. Positions and status are fixed enums; create/update input excludes `Loan` (that status is set by transfer flows).
5. ‚öÝÔ∏è **Ownership not enforced** ‚Äî all reads/writes look up the save by id only. See [3.7 Security #1](../3.7_Security.md#1-broken-object-level-authorization-idor--high).
6. Cache invalidation is partial: `invalidatePlayersCache` clears `players`/`players:active`(+`player:<id>`) but not `players:active:<season>` history. See [Tech Debt #4](../3.5_Technical_Debt.md).

## Extension Points
