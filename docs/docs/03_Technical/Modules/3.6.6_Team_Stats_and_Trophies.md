# 3.6.6 Team Stats and Trophies

## Responsibility

Two closely related concerns: **team season statistics** (per competition, per season) and **trophies** (a competition won by a stint in a year). Both hang off `ClubStint` and are largely auto-created by the save lifecycle, then edited or added manually.

## Files

```
features/team-stats/
├── team-stats.routes.ts
├── team-stats.controller.ts
└── team-stats.service.ts    listTeamStats, updateTeamStats
features/trophies/
├── trophies.routes.ts
├── trophies.controller.ts
└── trophies.service.ts      listTrophies, createTrophy, deleteTrophy
```

## Endpoints

| Method | Path | Notes |
|---|---|---|
| GET | `/api/saves/:saveId/team-stats` | `?season=current` / `?season=YYYY/YY`; none = all seasons & clubs. |
| PATCH | `/api/saves/:saveId/team-stats/:statsId` | `leaguePosition` (leagues) / `cupResult` (cups/European). |
| GET | `/api/saves/:saveId/trophies` | With club + competition. |
| POST | `/api/saves/:saveId/trophies` | `{ competitionId, year }` → attaches to current stint. |
| DELETE | `/api/saves/:saveId/trophies/:id` | — |

## Data Flow

### Team stats
- Rows are created automatically: on save creation (per country competition) and on season advance (per competition). The API does not expose a create-stats endpoint — you only `PATCH` existing rows.
- `updateTeamStats` validates `leaguePosition ≥ 1` and `cupResult` against the enum, confirms the row belongs to the save (`clubStint: { saveId }`), updates, and invalidates `team-stats[:season][:current]`.
- Listing filters out legacy rows without a `competitionId`.

### Trophies
- **Auto**: season advance upserts a `Trophy` for each ending-season competition the club won (league `leaguePosition === 1`, cup `cupResult === 'Campeao'`), keyed `(stint, competition, year)`.
- **Manual**: `createTrophy` attaches to the **current** stint, requires a valid `competitionId`, and is bounded by `@@unique([clubStintId, competitionId, year])`.

## Local Domain Rules

1. Team stats uniqueness: `@@unique([clubStintId, competitionId, season])`.
2. League competitions use `leaguePosition`; cups/European use `cupResult` (enum: `Campeao, Final, Semifinal, Quartas, OitavasOuFaseDeGrupos, Eliminado, NaoParticipou`).
3. Trophies uniqueness: `@@unique([clubStintId, competitionId, year])` — a competition is lifted at most once per stint per year.
4. Auto-trophies on season advance derive from the ending season's team-stats results.
5. Save ownership is confirmed before the operation (`assertSaveAccess` pattern).
6. `TeamSeasonStats` retains legacy nullable columns (`europeanCupResult`, `nationalCupResult`, `cupResult`); current code uses `competitionId` + `cupResult`.

## Extension Points

- A standings/season-summary aggregate endpoint (the MCP `get_season_performance` tool already composes this for chat).
- Bulk stat entry across competitions.
- Drop the dead `TeamSeasonStats` columns via migration.
