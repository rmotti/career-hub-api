# 3.6.3 Club Stints

## Responsibility

A `ClubStint` is one spell at one club within a save. This module owns listing the trail of clubs, reading the current club, and the **change-club** operation — the second most consequential transaction in the system after season advance.

## Files

```
features/club-stints/

└── club-stints.service.ts   listClubStints, getCurrentClubStint, createClubStint, updateClubStint
```

## Endpoints

| Method | Path | Notes |
|---|---|---|
| GET | `/api/saves/:saveId/club-stints` | All stints, oldest first. |
| GET | `/api/saves/:saveId/club-stints/current` | The `isCurrent` stint (reuses the list cache). |
| POST | `/api/saves/:saveId/club-stints` | `{ club, europeanCompetitionId? }` — change club. |
| PATCH | `/api/saves/:saveId/club-stints/:stintId` | Edit `club`/`startYear`/`endYear`. |

## Data Flow — Change Club (`createClubStint`)

In one transaction:
0. **Take a `pre-club-change` snapshot + `clubstint.change` audit** before any mutation (atomic with the change). Because step 4 detaches the **entire** squad, this is the only reliable undo — recover via the save snapshot restore (`POST /saves/:id/snapshots/:snapId/restore`).
1. Validate the new `club` (`clubExists`) and, if given, that `europeanCompetitionId` is a real `EuropeanCup` competition.
2. Resolve the new club's country → competition ids. If no European competition was supplied, **carry over** the current stint's European competition (so the team keeps its continental campaign).
3. Close the current stint: `isCurrent = false`, `endYear = currentYear`.
4. **Detach all players** of the old stint (`activeClubStintId = null`) — the squad does not follow the manager.
5. Create the new stint (`isCurrent`, `startYear = currentYear`).
6. Create `TeamSeasonStats` for the new stint (country competitions + European).

Then invalidate a broad set of keys: `club-stints`, `transfers[:current]`, `team-stats[:season][:current]`, `players[:active]` — because everything save-scoped changes shape when the club changes.

## Local Domain Rules

1. Exactly one current stint per save at any time.
2. Changing club is destructive to the active squad: players are released, not migrated. Re-acquiring them is a new transfer.
3. The European competition carries over unless explicitly overridden.
4. Save ownership is confirmed before the operation (`assertSaveAccess` pattern).

## Extension Points

- Optionally migrate selected players to the new club (a "bring your staff/players" flow).
- Per-stint summary endpoints (trophies, transfers, best season) for a career timeline.
- Enforce season/year ordering on `startYear`/`endYear` edits.

## Auto-audit

- Change-club transaction and cache invalidation read from `club-stints.service.ts` (`createClubStint`).
- European-competition carry-over confirmed in the `currentEuropeanStats` branch.
- Club, country, and competition resolution read from `clubs.service.ts` and `competitions.service.ts`.
