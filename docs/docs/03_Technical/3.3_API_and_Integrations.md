# 3.3 API Reference and Integrations

The canonical, always-current contract is the **Swagger UI at `/docs`** (generated from the Fastify route schemas). This file is the structured companion: transport rules, the error contract, the full endpoint inventory, and the external integrations.

## Transport

- **Base path**: every route is under `/api` except `POST /mcp` and the `GET /` → `/docs` redirect.
- **Auth**: `Authorization: Bearer <token>` (the `token` returned by sign-in/sign-up). Required for everything except `/api/auth/*` and `POST /mcp` (which authenticates itself).
- **Content type**: `application/json`. An empty body on a JSON request is parsed as `{}` (custom parser).
- **Compression**: gzip/br for responses over 1024 bytes.
- **CORS**: prod restricts to the trusted-origins list; dev allows all. `credentials: true`.

## Error Contract

Two envelope shapes:

```jsonc
{ "error": "Save não encontrado.", "statusCode": 404 }                       // generic
{ "error": "SHIRT_NUMBER_CONFLICT", "message": "O número 10 já está...", "statusCode": 409 }  // coded
```

| Status | When |
|---|---|
| 400 | validation (`Campo inválido: '<field>' — …`), bad reference (Prisma P2003), domain `AppError` |
| 401 | missing/invalid bearer token |
| 403 | (only if a role/plan guard is added — none today) |
| 404 | not found, **or** save owned by another user (no existence leak) |
| 409 | uniqueness conflict (Prisma P2002) or coded conflict (`SHIRT_NUMBER_CONFLICT`) |
| 429 | rate limit; carries `Retry-After` (seconds) |
| 500 | unhandled — `Erro interno do servidor.` |
| 502 | upstream (e.g., OpenAI returned no output) |

Error messages are **PT-BR**.

## Authorization Note (read before consuming)

`requireAuth()` proves the caller is authenticated, not that they own the `saveId`. **`saves`, `scout-playbooks`, `shortlist`, `saved-searches`, `scouting`** verify ownership; **`players`, `transfers`, `team-stats`, `club-stints`, `trophies`** do not (they 404 only if the save doesn't exist). No route is plan-gated. See [3.7 Security](3.7_Security.md).

## Integrations

### OpenAI Responses API (chat)
`features/chat/chat.service.ts` calls `openai.responses.create` with `model: gpt-4o-mini`, `instructions: COACH_PERSONA` (the "Junior" persona), the user's `message`, and one `mcp` tool pointing at `${mcpBaseUrl}/mcp` with the user's bearer token and `require_approval: 'never'`. `mcpBaseUrl` = `API_URL` or `<protocol>://<hostname>`. Continuation via `previous_response_id`. Requires `OPENAI_API_KEY`.

### MCP Server (self-hosted)
`POST /mcp` exposes read-only career tools to the OpenAI tool-use loop. See [Module 3.6.11](Modules/3.6.11_MCP_Server.md). Tools: `list_saves`, `get_active_save_context`, `get_finances`, `get_season_performance`, `analyze_squad_by_position`, `identify_squad_gaps`, `search_transfer_targets`, `evaluate_signing_fit`. Resources: `playbook://{saveId}`, `save://{saveId}/dossier`.

### Fit-score service (HTTP)
`shared/lib/fit-score-client.ts` POSTs candidate batches to `${FIT_SCORE_SERVICE_URL}/score/batch` (`{ club_name, position_group, objective, candidates[] }`) with a **3 s timeout**. On any error/timeout/unset URL it returns an empty map (fail-open → `null` fit scores). Results cached 6h.

## Environment Variables

| Variable | Purpose |
|---|---|
| `DATABASE_URL` | pooled Postgres (PgBouncer); `connection_limit=10&pool_timeout=20` appended at runtime |
| `DIRECT_URL` | un-pooled Postgres for migrations |
| `REDIS_URL` | Redis (defaults `redis://localhost:6379`) |
| `BETTER_AUTH_SECRET` / `BETTER_AUTH_URL` | Better Auth |
| `TRUSTED_ORIGINS` | comma-separated CORS/Better-Auth origins (merged with built-in wildcards) |
| `OPENAI_API_KEY` | chat |
| `FIT_SCORE_SERVICE_URL` | fit-score microservice |
| `API_URL` | base URL the MCP tool callback points at (else derived from the request) |
| `PORT` | listen port (default 3333) |
| `NODE_ENV` | toggles logging, cookie attributes, Prisma logging, singleton memoization |
| `LOG_LEVEL` | Pino level (default `info`) |
| `DISABLE_RATE_LIMIT` | `true` disables Better Auth's limiter (load tests only) |
| `ADMIN_USER_IDS` | (optional) admin allow-list for the Better Auth admin plugin |
| `VERCEL` | presence skips `app.listen()` (serverless) |
